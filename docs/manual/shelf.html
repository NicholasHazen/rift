Reference · The problem-first library

The shelf

Every external resource in the plan lives on this page, filed under the question it answers. Nothing here is homework. The shelf exists so that at the moment a question bites, the answer is already filed — and so no doc or video ever gets opened out of vague ambition. No question, no shelf space.

How an entry earns its place

Rule 1 of the operating system says never open a doc or a video without a written question it must answer. This page is that rule applied to the manual itself: the question comes first by construction, because the entry cannot exist without one. Why does problem-first beat curriculum-first? → doctrine-os.html#rule-1.

The entry template

Every entry carries three fields, in this order. An entry without a question is rejected at review.

  • The question — phrased as the moment of need ("My property silently never replicates — why?"), never as a topic.
  • Reach for it — the week or the trigger that makes the entry live.
  • Durability — one of the three classes below, plus the link itself.

Durability, and how this page stays true

Links rot at different speeds, so every entry declares its class. There are exactly three.

  • Durable repo — GitHub repositories, long-lived community sites, and vendor-hosted originals. They move rarely and break loudly; trust the bookmark.
  • Official-docs deep link — Epic's /documentation/ pages. They drift when Epic reshuffles the tree, so every entry's title doubles as its docs-search phrase — the search is the stable interface, the deep link is the convenience.
  • Search-query link — anything video-shaped or otherwise non-canonical. The link is a search, so it cannot rot; the question in hand does the filtering.

Maintenance law, for the fleet: new links land verified or search-query-shaped, nothing else. Epic /documentation/ URLs verify by fetch — and a fetch that returns a near-empty table-of-contents skeleton is a dead page, not a pass; Epic's docs site never serves an honest 404, so the skeleton is the 404. Epic /community/ URLs (and unrealengine.com itself) sit behind bot protection and never auto-verify, so nothing on this page links there directly — anything living behind that wall gets a search-query link instead. When a durable link dies, fix it here once. Every pull in the manual points at this page, so one edit heals them all.

And the pull budget: week pages carry at most three shelf pulls, concept pages close with two or three, and a pull is always the question plus a link to an anchor here — never a restated entry. If a week seems to need a fourth pull, it doesn't. The shelf keeps the rest.

Unreal MCP — the engine's own agent bridge

The question: "Can an agent reach into the running editor — and is that bridge load-bearing yet?" The answer as of 5.8: the bridge is first-party — the engine ships a ModelContextProtocol plugin (disabled by default, marked Experimental, loopback-only HTTP), and Epic publishes an official Claude Code plugin that wraps it in skills and toolsets, automation-test running included. Third-party MCP plugins predate this and are stale, commercially pivoted, or both — the first-party pair is the only path worth a timebox. Reach for it: week 0's experimental garnish, strictly timeboxed — it is garnish, not gate; its real value here is running tests and inspecting the world, never authoring Blueprint logic (the C++-for-systems rule doesn't bend for a robot). Durability: official-docs deep link — Unreal MCP · durable repo — github.com/EpicGames/unreal-engine-skills-for-claude-code-plugin.

The deferred tier — not yet, because

Everything below is good; that was never the question. Each entry waits behind a named trigger, because boarding early costs more than it teaches. When a trigger fires, the entry moves up into its topic section and gets a real question of its own.

  • Lyra. Not yet, because it is shipped-game scale and buries a week-2 learner. Trigger: week 8 or later, arriving with one specific question. Mine it for patterns; never a base. Official-docs deep link — the Lyra sample.
  • GASShooter. Not yet — same law as Lyra: mine, never a base. Trigger: stuck on a specific composed ability pattern that tranek describes and Narxim is too minimal to show. Durable repo — github.com/tranek/GASShooter.
  • Replication Graph. Not yet, because it is Fortnite-scale power at Fortnite-scale complexity. Trigger: a failed Friday perf gate that survived all six optimization passes and the entity cap — never speculatively. Official-docs deep link — Replication Graph.
  • Iris. Not yet, because nobody switches replication systems mid-project — production-ready as of 5.8, still opt-in, and the classic path remains the default. Trigger: Stage 2 at the earliest, evaluated fresh; the migration guide exists when that day comes. Official-docs deep link — Iris.
  • Mass. Not yet, because the slice's horde fits comfortably in actors under the entity cap. Trigger: an NPC ceiling the cap cannot hold — a Stage-3 conversation. Official-docs deep link — Overview of MassEntity.
  • Mover 2.0. Not yet, because it is experimental and the CharacterMovementComponent is load-bearing doctrine — prediction for free. Trigger: engine maturity plus a Stage-2 reason to move. Official-docs deep link — Mover.
  • Epic Online Services. The Stage-2 matchmaking signpost; nothing in Stage 1 needs it. Official-docs deep link — EOS documentation.
