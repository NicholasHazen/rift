S4 — The levers, one at a time

[NICK] One lever at a time, measuring between each, numbers into the repo.

Which of the six levers buys back the most milliseconds — ranked, committed next to the baseline, before any of them get pulled?

No answer key prints here, on purpose: the trace grades the ranking, and being wrong here is the lesson. Why a wrong prediction teaches more than a right one → concept-perf.html#lab

The six, in the order they get pulled — each one's move and landmine are printed where it lives:

  • Relevancy — attacks recipients.
  • Update frequency — attacks the clock.
  • Dormancy — attacks actor count; the strongest wire lever.
  • Anim tick — attacks the frame's anim slice. The caveat rides the handle: on a dedicated server nothing ever renders, so anim-graph gameplay silently dies there.
  • Crowd following — attacks the AI slice; pulled early, in S2, and measured here.
  • Minimal replication — attacks properties: set the NPC ASCs to Minimal mode.

Done when: six deltas sit next to the prediction in the repo, and the fattest remaining slice has a name.

What this week teaches

Almost nothing on this page, on purpose — this page is the lab notebook, and the theory lives on concept-perf.html, one page, worth reading in full before S3. Three orientations, one link each.

The cost model. Replication bandwidth is a product of four factors — actors, properties, update frequency, recipients — and the server's CPU stacks movement, AI, and animation beside the wire. Every lever this week attacks exactly one line of that bill, which is why they get pulled one at a time. How bad is the napkin math before anyone optimizes? → concept-perf.html#cost-model

Thirty-three milliseconds. The gate's number is one server tick at 30 Hz with headroom held back. What lives inside those 33 milliseconds, and which tenant is which? → concept-perf.html#frame-budget

The Friday protocol. From this week the gate is a six-step experiment — deploy, load, pressure, measure, commit, verdict — written so an agent runs every step that doesn't need hands. What exactly happens every Friday from now on? → concept-perf.html#gate-protocol

And one inoculation, taken early: the names the forums will offer when a Friday runs red — Replication Graph, Iris, Mass — are all real and all wrong for this project right now. Read the refusal before the first bad trace, not after → concept-perf.html#not-yet

One unshelving is sanctioned this week: Lyra, Epic's production-grade multiplayer sample. It stayed shelved until now for a reason — sample code reads as incantation until you've built the same machinery yourself, and by this week you have: replication, abilities, teams, all patterns you own. Mine it for team and replication patterns worth stealing, and log what you found — a one-line verdict on its shelf entry, so the next unshelving starts from evidence instead of a bookmark.
