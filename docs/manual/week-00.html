Chronicle fragment, run [unlogged] — The expedition hung tools in an empty workshop and named the mountain. Zero rocks were harvested.

By Friday you will have a UE 5.8 project running in PIE — Play In Editor, running the game inside the editor itself — with two clients and a dedicated server, meaning a headless server that no player sits behind. On top of that you will have a pushed repo and an agent that builds the code from the command line, and you will know where code lives in an Unreal project because you read it yourself. None of it is the game; all of it is the workshop, which is why this week gets the tightest box in the manual.

▣ Gate — definition of done

The character runs in PIE with 2 clients plus the dedicated-server option, because the week's test topology — the shape of clients and server you test under — is PIE + dedicated from the very first Play. The repo is pushed, and a fresh clone opens clean in the editor. An agent builds from the CLI (the command line) and correctly answers "where is player input handled in this project?" Clip #001 is posted. Setup week does not get an encore.

The week is an empty shop, and you are the one hanging tools. The only goal is a running project, a healthy repo, and an agent that can build the code — and a goal that modest is exactly why the box is hard. The trap this week: perfecting the workshop instead of using it. You'll know you're in it when editor settings are open before the gate passes. The move: you get oriented by needing things.

The project is OneMoreRock — the exit decision, the choice to extract or push deeper, in three words. The codename names the .uproject file and the primary C++ module, and it exists before the project does because renaming a UE project later is misery. The register carries the dated decision, and the decision stops there on purpose: it doesn't pre-commit the Bit, the horde's placeholder identity, which keeps its week-3 date.

What this week teaches

All five ideas here are orientation. Where an idea has a deeper home, its paragraph carries the one link, and the other two are already the whole week-0 lesson.

The two worlds. An Unreal project splits into Source/, where C++ schema lives, and Content/, where binary assets live — the split is why Git LFS exists and why agents can author content without ever touching the schema layer. Your template read-through is the first walk through both. Where exactly does everything live, and why does the split match how you already build software? → concept-anatomy.html#two-worlds

The build toolchain. UnrealBuildTool drives every build, and on this PC the entry point is Build.bat, exactly the one the tutorials name. Live Coding works too, which means Ctrl+Alt+F11 recompiles function bodies into the running editor, though header and class-layout changes still take the full close-build-relaunch cycle. Knowing that much is this week's whole requirement. The exact commands land in CLAUDE.md §Build the day the fleet proves them, so they're never folklore.

PIE test topologies. The Number-of-Players field and the dedicated-server option are the week's entire networking lesson: from the very first Play, there is a server, and it isn't you. Which test topology tells the truth about which bugs — and when does the convenient one lie? → concept-netcode.html#topologies

Enhanced Input. The Third Person template is built on Enhanced Input — mapping contexts and input actions rather than hard-wired key bindings — and week 2's ability binding routes through it. This week you only need to know it exists and recognize it in the template code you're reading.

The build host. One question this week seeds instead of answers: this PC develops the game fine, and it can even produce both of week 7's artifacts, the Win64 clients natively and the Linux dedicated server through Epic's Windows-hosted cross-compile toolchain. The catch is that both need a source-built engine, which is a disk-and-hours commitment. Whether the PC doubles as the build host or a separate box carries it is OQ-03, week-1 dated — and it takes your sign-off, not a fleet default. What does shipping need that daily development doesn't? → concept-shipping.html#build-host

⚠ Gotcha seeds
  • The Unreal Project Browser creates <Location>/<ProjectName>/ and errors if that folder already exists, so create at a temp location, quit, move the contents (.uproject, Source/, Content/, Config/) into the repo root, reopen, and answer Yes to the "rebuild missing modules" prompt — it recompiles from Source/ and opens clean. If it bites differently, it's the first entry in UE-GOTCHAS.md.
  • No Visual Studio 2022 with the Game development with C++ workload → C++ project creation fails, and the fix is a 10+ GB install you don't want to discover at the create step.
  • UnrealBuildTool, not Visual Studio, owns the toolchain pick: a fresh VS update can outrun what the engine blesses, and UBT says so as a warning about an unsupported MSVC toolset. Install the exact toolset it names via the VS Installer's Individual Components instead of debugging a too-new compiler's errors.
  • First open of a fresh C++ project compiles the module, then thousands of shaders — 15–30 minutes of progress bars is normal, not a hang. Start it, then make the IDE call while it churns.
  • compile_commands.json goes stale after module or header changes — re-run the generate command, or the fleet navigates last week's code.
  • Do not source-build the engine this week — the launcher binary is the point. Source is week 7's problem, on whatever hardware answers OQ-03.
