Camp as machine

Week 05Phase: CampBudget: 5 sessions + a stretch spikeExam: join-in-progressresolution: full
Chronicle fragment, run [unlogged] — Day 4. The wall held for eleven minutes. The extractor never stopped ringing.

By Friday you will have two clients co-building a camp whose extractor harvests unattended while a wave gnaws at the walls — and a third client that joins mid-session and sees all of it correctly. You will understand the two problems basebuilding drags into multiplayer — trust and late joiners — well enough to design for the first and run a deliberate exam on the second.

▣ Gate — definition of done

Two clients co-build a camp, the extractor pulls unattended, and a wave damages walls; then a third client joining mid-session sees a correct world. Topology: PIE (Play In Editor — the game run inside the editor) with the dedicated-server checkbox, which stands up a headless server, and net emulation on per rule 5, so every test carries simulated latency.

The vision says the camp isn't a home. It's gear: the machine the expedition assembles to convert a hostile rift into a haul. The machine's footprint is one hand on the greed dial, the risk-for-reward lever this whole slice exists to test. This week the machine gets its parts: a ghost, a wall, an extractor, a crate. Trust gets designed in from the first trace — the aiming ray a placement starts with — because a wall is the biggest thing a client has asked the server to believe so far. Late joiners get their exam on Friday, deliberately, while the camp is small and the fixing is cheap.

The sessions

The week runs five sessions and one sanctioned stretch. The first two put trust into placement, the middle two build the machine and the siege that answers it, and the fifth is the exam. Done-marks land as sessions close, per the week-0 convention. One errand rides the week: find out which platforms your friends actually play on. Week 6's census runs this week so that a Mac friend surfaces now, not on friend night — a Mac client is the one build this Windows PC can't package. What else does friend night need in place before it arrives? → week-06.html#pre-flight.

S1 — Ghost

[NICK] You build the placement component's client half. A trace from the camera picks the spot, a ghost mesh snaps onto it, and a validity tint delivers the verdict: green where the placement is legal, red plus a game-show buzzer where it isn't. The ghost is the client imagining — instant, cosmetic, and the server never hears about it. That split is the week's first lesson, built before it's taught. Done when: the ghost previews and tints instantly on a client, and no placement traffic has crossed the wire yet.

[FLEET] The fleet routes the structure numbers — wall and extractor costs, footprints, scan radius, tick rate — down the week-4 tuning path as JSON. You review the schema once, and the values stay in text after that. Done when: the file imports and a wall knows its price without anyone recompiling.

S2 — Trust

[NICK — by hand] Server_PlaceStructure is this session's centerpiece. It revalidates everything the ghost claimed — range, overlap, cost — and only then spends the resources and spawns the replicated structure, the copy the server now sends to every client. The one-line authority read: the client proposes a transform (a position and rotation), the server validates it, and only the server spawns. Structure count reports to the ire subsystem, the server-side service that banks the rift's anger, so the machine's footprint is on the dial from the first wall. Then run the cheater test on the new verb the week it's born — forge the worst placement you can and watch it die on the server. What does the test prove, and how is it run? → concept-netcode.html#cheater-test. Done when: a legal wall stands on every client, and every illegal placement you can forge fails safe.

S3 — The greed machine

[NICK] You build the extractor, the camp's reason to exist. It scans the nearby resource nodes, auto-harvests them on a timer, and generates ire every minute it runs. Installing one is pushing the dial with both hands — make sure the flavor text says so. Why does automated harvest price violence faster than a pick ever could? → concept-ire.html#economy. Done when: an extractor placed by one client pulls unattended, and the ire stage climbs while both players stand perfectly still.

[FLEET] The fleet gives the extractor its voice by wiring install and tick lines from the word lists through the JuiceKit, the game's one shared feedback component. The payload box below says what it must sound like. Done when: an extractor tick makes a sound and a number on every client near it.

S4 — The wall

[NICK] Wall HP arrives through GAS (the Gameplay Ability System, the engine's stats-and-abilities framework), because structures get attributes too — the ledger pays again. Register the wall as a dynamic nav obstacle — a live entry in the NavMesh, the engine's map of walkable ground — so paths bend around fresh construction. Then widen NPC targeting to nearest-player-or-structure, so the horde gnaws what it can't path around. Walls block and are gnawable, which is standing law, so mazes will be degenerate for a while. Let playtesters find that and enjoy it before you fix it. Funnels and kill-boxes are now real play, which is exactly the tower-defense legibility the physics doctrine promised — and nobody will need it explained. Done when: a wave gnaws a wall down through its damage stages while paths bend around the walls still standing.

[FLEET] The fleet dresses the gray-box wall — the plain placeholder version — in its damage-stage looks. Tint shifts and cracks stand in for art, so the wall's health is readable at a glance from across the camp. Done when: a glance tells you which wall the horde is winning against.

S5 — The exam

[NICK — by hand] Build the storage box first. It is a shared crate for chunks — the harvested rock itself — and one more replicated structure stamped from the S2 pattern. Then comes the join-in-progress exam: build a camp with two clients, connect a third cold, and walk the written audit. The checklist lives at concept-netcode.html#join-in-progress, and it is a procedure, not a vibe pass. Fix everything the audit catches, and write every fix into UE-GOTCHAS.md. Done when: the third client's world diffs clean against the server's — the gate's last condition.

Stretch — the sanctioned Tier-3 spike

[NICK — one session, hard timebox] The spike is a single server-simulated boulder you can shove downhill into the horde, run under the standing 100 ms of net emulation. It is the one Tier-3 spike the physics budget sanctions, where Tier 3 means free physics bodies networked as gameplay. The spike comes with homework so the verdict is informed, not vibes, and both pre-reads sit on the shelf below. If the boulder sings, log it as a future rift-modifier ingredient; if it fights you, that's the doctrine confirmed at the cost of one evening. Either way it does not enter the Stage-1 critical path; the ambitions parked behind the verdict live at OQ-19. Why is a free rigid body under latency a different sport from everything else this week builds? → doctrine-physics.html#tier-test. Done when: the timebox expires with a dated one-line verdict — sings or fights — recorded against OQ-19.

The trap this week: the spike leaking. You'll know you're in it when the boulder is negotiating for a second evening. The move: the timebox is the verdict, because a boulder that needs more time to feel good has already answered "fights."

⚑ Fun payload — the machine sings

Player: the extractor cha-chings on every tick — it is a machine that prints money and the rift's anger simultaneously, and it should sound like both. Walls crack visibly by damage stage, and invalid placement gets a game-show buzzer. Builder: the goon-vs-wall moment. The first time the horde swarms a wall and starts gnawing it while your extractor keeps ringing behind you — that's the game's whole thesis in one screenshot, and Clip #006.

What this week teaches

Four ideas follow, and each gets only an orienting pass with a single link, because the deep homes hold the answers.

Trust, at placement scale. Placement is the DM model applied to construction. The ghost preview is client-side, which means it is instant, cosmetic, and purely the client imagining. The real placement is a Server RPC — a client-to-server request — that the server validates for range, overlap, and cost before anything is spent. Never trust the client's opinion of where a wall goes; a modified client will try to place walls inside spawners. Why does every verb that changes the world route through the server? → concept-netcode.html#authority

The carpet, changed at runtime. Walls change the walkable carpet mid-session, and the engine offers two ways to keep the NavMesh honest — priced very differently. What does it cost to change the carpet at runtime, and which mode do walls want? → concept-horde.html#navmesh

The join-in-progress exam. A client connecting after the camp exists must receive the whole picture from replication alone. At 24 players with rolling arrivals, this bug class is fatal, which is why the exam runs now, while it's cheap. And it runs predict-then-verify, because a prediction graded is worth ten answers handed over.

Before S5's third client connects, write your prediction down: what exactly will it see wrong, and why?

Every miss on your list will have the same shape: a fact that lives somewhere other than a replicated property — a value the server keeps synced across clients. It was set once, on the clients that happened to be present, and remembered nowhere the server replicates. State must live in replicated properties, not in the memory of having-been-there. That law headlines the full failure taxonomy and the audit checklist at the theory page — grade your prediction against it after the exam. What should you have predicted, and how do you audit for the rest? → concept-netcode.html#join-in-progress

The ledger pays again. A wall's HP is an AttributeSet, the GAS bundle that holds a thing's stats, and the horde's gnawing is a transaction against it. That is the same ledger that already moves player health, now paying its week-5 dividend. What else does the ledger pay for, and when? → concept-gas.html#dividends
