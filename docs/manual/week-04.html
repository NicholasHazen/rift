The heart: ire and the director

Week 04Phase: Core loopBudget: 5 sessionsFun engine unlock: Director's Remoteresolution: full
Chronicle fragment, run [unlogged] — The party idled for three minutes and drew two geese. The party gorged for three minutes and drew nineteen. The rift itemized the difference.

You take your hands off the keyboard for three minutes, and the rift sends two bored geese. Then you harvest like the vein closes at midnight, and the pool fattens on every tick while the meter walks its stages. By the time the words read The rift is FURIOUS. the ridge is a honking wedge three waves deep — and every goose in it is a bill you ran up yourself. Clip #005 is rolling.

By Friday you will have a rift that answers greed with violence — three greedy minutes buy visibly bigger waves than three idle ones, retuned from a text file with no recompile. And you will understand the machine that does it: a wallet, a spender, and a meter that speaks.

▣ Gate — definition of done

Demonstrable causality: greed for three minutes produces visibly bigger waves than idling — in PIE with the dedicated-server checkbox, under the net emulation that has been on permanently since week 3. Retuned from JSON with no recompile. The Remote works. Record this clip carefully — it's the first artifact of the actual game.

This is the week the manual exists for. Everything so far was parts — a node that breaks, a ledger that settles, a horde that walks; this week is the idea: the greed dial becomes code. It lands deliberately early, partly because every later system tunes against it, and mostly because from Friday onward you're not doing engine exercises anymore — you're building your game. That distinction is fuel, and you need the tank full for the eight weeks that remain.

One prerequisite listen, same as week 1's: before S1, hear the dungeon-master-with-a-wallet page through once, start to finish. It is a listen, not a session — it holds the economy, its Unreal shapes, the spawn table, the cap, and the lineage, and this week is its lab.

The trap this week: building Valve's director instead of yours. You'll know you're in it when the spend loop grows a pacing state machine before the causality demo has run once. The move: ship the deliberately dumb loop, run the ritual, and let the JSON do the escalating — the deck on the shelf is for stealing the skeleton, not the org chart. Tune, don't add.

The sessions

Five sessions: a wallet, a schema, a spender, a voice, and the controls. Each carries its done-when, per the week-0 convention; done-marks land as sessions close.

S1 — The pool

[NICK — by hand] Hand-write UIreSubsystem — rule 4, and this week carries the last firsts for a while: the subsystem here, the codebase's first DataAsset in S2. Give it accrual functions for the three mint families — harvest ticks, kills, structure-minutes — called via a small event bus or direct calls; direct is fine at this size. The pool itself stays server-side, and the current value mirrors onto GameState as an ordinary replicated property whose one-line authority read writes itself: the server mints and writes, clients only read. Why GameState and not PlayerState? → concept-ire.html#shapes. Done when: you set ire-per-kill to a number you'll recognize, put Gerald down, and the mirror jumps by exactly that much on both clients.

S2 — The knobs

[NICK — by hand] URiftTuningDataAsset and the import path — the project's first DataAsset, so the class is yours. The mechanism is a split that decides what stays cheap forever: scalars and curves — ire per harvest tick, per kill, per structure-minute, the wave budget curves — ride runtime JSON, reloaded live by ire.ReloadTuning, the first Remote command, arriving three sessions before the Remote. Anything that names an asset — spawn-table rows pointing at NPC classes — rides DataTables and accepts the cook. That split is what makes S5's retune-without-recompiling true tonight and still true on the packaged VPS from week 7 on. Why two pipelines instead of one? → concept-anatomy.html#tuning-as-text

[FLEET] The first tuning file, fleet-authored the day the schema stands; you review it like it's an EMURJ metafield spec, because it basically is. Done when: you edit a rate in the JSON, run ire.ReloadTuning, and the next mint uses the new number — no recompile, no cook.

S3 — The spender

[NICK] DirectorComponent: every N seconds, convert the accumulated pool into a wave budget and buy goons from the spawn table until it's spent. The loop is deliberately dumb and stays that way — escalation comes from the tuning curves, not from cleverness in the C++. Week 3's WaveSpawner was the tap; the director takes over the faucet, and the plumbing underneath doesn't change. What are the rows it shops — costs, weights, tiers? → concept-ire.html#spawn-tables. Done when: waves arrive sized by the pool instead of by a constant — a fat pool buys a visibly bigger wave, in PIE + dedicated.

[FLEET] Spawn-table rows in S2's schema — the goon plus week 3's second variant, costed and weighted, so the director has something to choose between.

S4 — The voice

[NICK] Debug ire meter plus the escalating flavor stages, wired through the JuiceKit — stage transitions get a sound. The stage strings live in the payload box below and ship exactly as written; the meter reads the GameState mirror, never the subsystem. Done when: crossing a stage threshold changes the text on every client and fires its stinger.

[FLEET] Stinger wiring through the established JuiceKit hooks, and the stage thresholds into the tuning file — boundaries are scalars, so they hot-reload like everything else.

S5 — The Remote

[NICK] Assemble the god panel: console commands first — ire.Give 500, ire.ForceWave, ire.SpawnElite, KillAllGeese — debug UMG only if it fits inside the evening. Then the ritual demo, which is the gate: idle three minutes, note the waves; harvest greedily three minutes, note the waves. If the difference isn't visceral, tune the JSON until it is — without recompiling, which is the whole point. Done when: the causality is undeniable on camera and Clip #005 is posted.

[FLEET] Command implementations drafted from the ire.ReloadTuning pattern, each logged onto the accumulating command list the day it's born.

What this week teaches

Five ideas, all orientation, one link each — the system's teaching home is concept-ire.html.

The wallet. Ire is currency players mint by being greedy — every harvest tick, every kill, every structure standing — and the director spends that currency on violence: difficulty stops being a setting and becomes a consequence. Minting runs on the players' clock; the answer arrives on the world's. How does greed become geese, box by box? → concept-ire.html#economy

The shared pool. Ire pools expedition-wide — one number for the whole table, standing law with a date on it. Why is a shared pool a feature instead of a griefing vector? → register.html#d-ire-pool

The shapes. UIreSubsystem is a World Subsystem — an engine-managed, world-scoped singleton, server-only — with the pool's value mirrored onto GameState for clients; DirectorComponent is the spender beside it. Which shape owns which job, and what may a client never read directly? → concept-ire.html#shapes

Tuning as text. Every tuning value this week — rates, curves, costs, weights, the wake interval — lives in text, never in a constant: rule 6 at full power, and from this week forward your agents author game balance as files. Your proc-gen belief just grew hands. By what path does a text edit become live tuning, in PIE and on a packaged server? → concept-anatomy.html#tuning-as-text

The lineage. The spend loop has ancestors — Left 4 Dead's director, Deep Rock's swarm pacing, Risk of Rain's time-scaling — and one deliberate difference: theirs mint their own budgets from time and intensity, yours is minted by the players. Steal the pacing skeleton; keep the twist. When does the canonical director relent, and which knobs did Valve expose? → concept-ire.html#lineage
