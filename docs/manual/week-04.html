The heart: ire and the director

Week 04Phase: Core loopBudget: 5 sessionsFun engine unlock: Director's Remoteresolution: full
Chronicle fragment, run [unlogged] — The party idled for three minutes and drew two geese. The party gorged for three minutes and drew nineteen. The rift itemized the difference.

You take your hands off the keyboard for three minutes, and the rift sends two bored geese. Then you harvest like the vein closes at midnight, and the pool fattens on every tick while the meter walks its stages. By the time the words read The rift is FURIOUS. the ridge is a honking wedge three waves deep — and every goose in it is a bill you ran up yourself. Clip #005 is rolling.

By Friday you will have a rift that answers greed with violence — three greedy minutes will buy visibly bigger waves than three idle ones, and you will retune the difference from a text file with no recompile. And you will understand the machine that does it: a wallet, a spender, and a meter that speaks.

▣ Gate — definition of done

Demonstrable causality: greed for three minutes produces visibly bigger waves than idling for three. The demo runs in PIE (Play In Editor — running the game inside the editor) with the dedicated-server checkbox on, so a proper headless server owns the world. The net emulation that has been on permanently since week 3 — simulated lag and packet loss — stays on. The retune comes from JSON with no recompile, and the Remote works. Record this clip carefully — it's the first artifact of the actual game.

This is the week the manual exists for. Everything so far was parts — a node that breaks, a ledger that settles, a horde that walks. This week is the idea itself: the greed dial becomes code. It lands deliberately early, partly because every later system tunes against it, and mostly because from Friday onward you're not doing engine exercises anymore — you're building your game. That distinction is fuel, and you need the tank full for the eight weeks that remain.

There's one prerequisite listen this week, same as week 1's: before S1, hear the dungeon-master-with-a-wallet page through once, start to finish. It is a listen, not a session — it holds the economy, its Unreal shapes, the spawn table, the cap, and the lineage, and this week is its lab.

The trap this week: building Valve's director instead of yours. You'll know you're in it when the spend loop grows a pacing state machine before the causality demo has run once. The move: ship the deliberately dumb loop, run the ritual, and let the JSON do the escalating — the deck on the shelf is for stealing the skeleton, not the org chart. Tune, don't add.

The sessions

Five sessions carry the week: a wallet, a schema, a spender, a voice, and the controls. Each carries its done-when, per the week-0 convention, and the done-marks land as sessions close.

S1 — The pool

[NICK — by hand] Hand-write UIreSubsystem, the server-side wallet — that's rule 4 in force. This week carries the last of the firsts for a while: the subsystem here, and the codebase's first DataAsset, a saved config object, in S2. Give it accrual functions for the three mint families — harvest ticks, kills, and structure-minutes — and call them through a small event bus or with direct calls, since direct is fine at this size. The pool itself stays server-side, and its current value mirrors onto GameState, the match's shared replicated state, for clients to read. It rides there as an ordinary replicated property, and its one-line authority read writes itself: the server mints and writes, and clients only read. Why GameState and not PlayerState? → concept-ire.html#shapes. Done when: you set ire-per-kill to a number you'll recognize, put Gerald down, and the mirror jumps by exactly that much on both clients.

S2 — The knobs

[NICK — by hand] Hand-write URiftTuningDataAsset and its import path. It's the project's first DataAsset, so the class is yours. The mechanism is a split that decides what stays cheap forever. The scalars and curves — ire per harvest tick, per kill, per structure-minute, plus the wave-budget curves — ride runtime JSON, which ire.ReloadTuning reloads live. That command is the first Remote command, and it arrives three full sessions before the Remote panel itself. Anything that names an asset rides DataTables instead — the spreadsheet-shaped rows that point at real content, like the spawn-table rows pointing at NPC classes. Those rows accept the cook, Unreal's asset-packaging build step, because a named asset reference can't live in a loose text file. That split is what makes S5's retune-without-recompiling true tonight and still true on the packaged VPS — the rented server the game ships to — from week 7 on. Why two pipelines instead of one? → concept-anatomy.html#tuning-as-text

[FLEET] The fleet authors the first tuning file the day the schema stands, and you review it like it's an EMURJ metafield spec, because it basically is. Done when: you edit a rate in the JSON, run ire.ReloadTuning, and the next mint uses the new number — no recompile, no cook.

S3 — The spender

[NICK] Write DirectorComponent, the pool's spender. Every N seconds it converts the accumulated pool into a wave budget and buys goons from the spawn table until that budget is spent. The loop is deliberately dumb and stays that way, because the escalation comes from the tuning curves rather than from cleverness in the C++. Week 3's WaveSpawner was the tap, and now the director takes over the faucet while the plumbing underneath stays the same. What are the rows it shops — costs, weights, tiers? → concept-ire.html#spawn-tables. Done when: waves arrive sized by the pool instead of by a constant — a fat pool buys a visibly bigger wave, in PIE + dedicated.

[FLEET] The fleet fills in the spawn-table rows in S2's schema — the goon plus week 3's second variant, both costed and weighted — so the director has something to choose between.

S4 — The voice

[NICK] Build the debug ire meter and the escalating flavor stages, both wired through the JuiceKit — the game's feedback layer — so every stage transition earns a sound. The stage strings live in the payload box below and ship exactly as written. The meter reads the GameState mirror, never the subsystem. Done when: crossing a stage threshold changes the text on every client and fires its stinger.

[FLEET] The fleet wires the stingers through the established JuiceKit hooks and drops the stage thresholds into the tuning file — those boundaries are scalars, so they hot-reload like everything else.

S5 — The Remote

[NICK] Assemble the god panel. The console commands come first — ire.Give 500, ire.ForceWave, ire.SpawnElite, KillAllGeese — and the debug UMG, Unreal's UI toolkit, only if it fits inside the evening. Then comes the ritual demo, which is the gate: idle for three minutes and note the waves, then harvest greedily for three minutes and note the waves. If the difference isn't visceral, tune the JSON until it is — without recompiling, which is the whole point. Done when: the causality is undeniable on camera and Clip #005 is posted.

[FLEET] The fleet drafts the command implementations from the ire.ReloadTuning pattern and logs each one onto the accumulating command list the day it's born.

⚑ Fun payload — the Director's Remote

Builder: a god panel, built from debug UMG or console commands — ire.Give 500, ire.ForceWave, ire.SpawnElite, KillAllGeese. It's test tooling that happens to be the most fun object in the project — you get to play DM over your own world, and in week 11 you'll do it live on your friends, the single most memorable playtest trick a solo dev has. It's fun engine #2, and its command list accumulates in exactly one place → fun-engines.html#remote. Player: the ire meter speaks. Instead of just a number, it climbs through escalating flavor stages: The rift stirs. → The rift notices you. → The rift is FURIOUS. Cheap text, real dread. It quietly opens a design door, too, because mystery is one of your load-bearing beliefs. Whether the stages ever give way to exact numbers is OQ-05's to answer.

What this week teaches

Five ideas follow, each one orientation with a single link, because the system's full teaching home is concept-ire.html.

The wallet. Ire is the currency players mint by being greedy — every harvest tick, every kill, every structure left standing — and the director spends that currency on violence. Difficulty stops being a setting and becomes a consequence. Minting runs on the players' clock; the answer arrives on the world's. How does greed become geese, box by box? → concept-ire.html#economy

The shared pool. Ire pools expedition-wide, which means one number for the whole table. That pooling is standing law now, with a date on it. Why is a shared pool a feature instead of a griefing vector? → register.html#d-ire-pool

The shapes. UIreSubsystem is a World Subsystem — an engine-managed, world-scoped singleton that runs server-only — and the pool's value mirrors onto GameState for clients to read. DirectorComponent is the spender that sits beside it. Which shape owns which job, and what may a client never read directly? → concept-ire.html#shapes

Tuning as text. Every tuning value this week — rates, curves, costs, weights, the wake interval — lives in text, never in a constant. That's rule 6 at full power, and from this week forward your agents author game balance as files. Your proc-gen belief just grew hands. By what path does a text edit become live tuning, in PIE and on a packaged server? → concept-anatomy.html#tuning-as-text

The lineage. The spend loop has ancestors — Left 4 Dead's director, Deep Rock's swarm pacing, Risk of Rain's time-scaling — and one deliberate difference. Theirs mint their own budgets from time and intensity, but yours is minted by the players. Steal the pacing skeleton; keep the twist. When does the canonical director relent, and which knobs did Valve expose? → concept-ire.html#lineage
