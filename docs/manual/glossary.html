Glossary

When to (re)read this: the first evening back after a gap, when the words have gone soft — or any time a page uses a term like it's obvious and it isn't.

Every term the manual leans on: one sentence each, and a link to the page that actually teaches it. This is vocabulary insurance — five minutes here and the whole working vocabulary is back. The glossary points; it never teaches.

Two rules keep this page honest. An entry is one spoken sentence plus one teaching link — an entry without a link is incomplete, because a glossary that starts teaching becomes a second copy of every page it serves. And across the wiki, the first use of a taught term on any page links either here or straight to the term's teaching anchor, so no page ever strands you on a word.

The director — the server-side spender that wakes on a timer, converts pooled ire into a wave budget, and shops the spawn table until the budget runs out. → concept-ire.html#economy

The Director's Remote — the god panel: console commands for tuning the live game and for playing human director during playtests. → fun-engines.html#remote

The Chronicle — the end-of-run LitRPG recap rendered from the session's event log: string templates today, local-LLM prose later. → fun-engines.html#chronicle

Gerald — the target dummy: a nameplate, a scream, and a career. → fun-engines.html#bestiary

The craft's words

Unreal's vocabulary, plus the engineering words this project coined — the terms the concept pages exist to teach.

PIE — Play In Editor: the editor's built-in play mode, launching N client windows plus, when asked, a headless dedicated server — the everyday test rig. → concept-netcode.html#topologies

GAS — the Gameplay Ability System: Epic's rules engine for abilities, attributes, and effects — the ledger every stat change posts to. → concept-gas.html

CMC — the CharacterMovementComponent: the engine's battle-tested predicted mover, the reason walking and carrying feel instant under lag for free. → doctrine-physics.html#cmc

UMG — Unreal Motion Graphics, the engine's widget UI system: the HUD's raw material, driven from C++ per rule 6. → shelf.html#benui

Authority — the server is the DM: it owns every fact in the world, and clients send requests, never results. → concept-netcode.html#authority

Ownership — the private chain of actors a connection may speak through, which decides where RPCs are allowed to run. → concept-netcode.html#ownership

Replication — the one-way conveyor that carries the server's facts down to every client's copy of the world. → concept-netcode.html#facts-vs-moments

RepNotify — the client-side callback that fires when a replicated property's new value lands — coalesced and unordered, so it must state facts, never tell stories. → concept-netcode.html#repnotify

Relevancy — the lever that stops shipping an actor to clients too far away to care — and on those clients the actor doesn't update less, it doesn't exist. → concept-perf.html#passes

Dormancy — the lever that removes a sleeping actor from replication entirely, until FlushNetDormancy() puts it back in the conversation. → concept-perf.html#passes

Join-in-progress (JIP) — the late joiner's world, rebuilt from replicated facts alone — the reason a moment may never be the only carrier of state. → concept-netcode.html#join-in-progress

Listen vs dedicated — the two server shapes: a player's machine that also hosts, versus a headless process that is nobody's client. → concept-netcode.html#topologies

ASC — the AbilitySystemComponent, the account that makes an actor a citizen of the rules engine: no account, no citizenship. → concept-gas.html#asc-placement

GameplayEffect — the only legal way to change an attribute: a transaction posted to the ledger, never a direct write. → concept-gas.html#ledger

GameplayTag — a hierarchical adjective — State.Stunned, Damage.Type.Storm — that abilities, effects, and checks all speak as one shared vocabulary. → concept-gas.html#tags

StateTree — Epic's flowchart brain: states with enter conditions, tasks, and transitions, re-selected from the root whenever an exit fires. → concept-horde.html#statetree

NavMesh — the baked map of walkable surface that NPC pathfinding runs on, rebuilt dynamically where players wall it off. → concept-horde.html#navmesh

Cook — the asset-compile stage: every .uasset baked from editor format into the target platform's optimized format, by walking the reference graph. → concept-shipping.html#pipeline

Package — the zip stage: cooked files packed into .pak archives and bundled with the binaries into something shippable. → concept-shipping.html#pipeline

The chunk contract — pop → settle → sleep → net-dormant: the four words every loose body in the game signs. → doctrine-physics.html#chunk-contract

Tier 1 / 2 / 3 — the physics budget's three price points: cosmetic and free, character-collision and cheap, true rigid bodies and quarantined. → doctrine-physics.html#tier-test

The cheater test — the procedure run on every server-authoritative verb the week it ships: lie to the server as a client and confirm the lie bounces. → concept-netcode.html#cheater-test
