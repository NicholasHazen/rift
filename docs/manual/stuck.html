The 90-minute valve

Minute 91, and the problem is still grinding. This page asks for no judgment — everything on it was decided in advance. Read the three exits and take one.

The three exits

In order. Each one is the session continuing. The wall is the only thing that gets no more time tonight.

  1. Hand the wall to an agent — as a lecture, not a fix. Ask for the failing subsystem explained end-to-end: where it starts, what it owns, where your code enters it. A minute-91 wall is usually a wrong model, not a wrong line. Twenty minutes of being taught beats sixty more of being lost.
  2. Post the wall in Unreal Source — its address is below, filed by wall type. Write it while the context is hot: what you expected, what happened, the smallest code that shows the gap. Then close the tab. Answers arrive on other people's clocks — the post is the exit, not the reply.
  3. Switch to this week's parallel task. It's in the box below, already chosen — because choosing a fallback while stuck is itself a task, and it won't happen at minute 91.

If the wall has a name

Exits one and two sharpen when the wall has a shape. Three shapes recur, and each has a standing address on the shelf, filed under the question it answers.

  • "How do experienced UE netcode developers solve the mid-tier multiplayer problems the docs skip?" → Vorixo's devtricks, plus Forsythe's replication video — for netcode walls: the property that won't replicate, the RPC that silently does nothing.
  • "How does GAS actually behave under the hood when tranek and the docs both run out?" → The Games Dev (KaosSpectrum) — for GAS walls: the ability that misfires, the attribute that lies.
  • "Where does the valve vent when the community has to answer?" → Unreal Source — for every other wall. This is where exit two posts.

The doctrine behind this page is rule 3 of the operating system: one wall must never eat a session, because momentum is the resource the whole plan runs on. Whichever exit you take, the close doesn't change — commit what's honest, write the tomorrow-note, done. Why is the valve a rule instead of a preference? → doctrine-os.html#rule-3. Walls keep.
