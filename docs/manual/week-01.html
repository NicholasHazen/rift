The humble resource node

Week 01Phase: Core loopBudget: 5 sessionsresolution: full
Chronicle fragment, run [unlogged] — First vein depleted; one chunk carried to the crate at a waddle. Total haul: one rock. The award for Heaviest Load went uncontested.

By Friday you will have a resource node that two networked clients can harvest, break, and haul — and you will have felt the dungeon-master model work instead of reading about it. Everything in the rift — ire, haul, the greed dial, the exit decision itself — is downstream of one primitive, a thing in the world players extract value from, and this week that primitive forces the entire networking core through your hands.

▣ Gate — definition of done

Two PIE clients plus the dedicated-server checkbox: both see the same node break, the same chunk pop, and a carried chunk deposit correctly — server-authoritative end to end, including a client that grabs a chunk another client dropped. Carrying visibly slows you. The cheater test fails safe. Harvesting already feels good. Clip #002 posted.

One prerequisite listen, then five build sessions. Before S1, hear the dungeon-master page through once, start to finish — it is the longest listen in the manual, this week is its lab, and every session below links back into it by anchor.

S1 and S2 are yours alone: the first replicated property and the first RPC of the codebase are first-of-pattern, and first-of-pattern belongs to your hands (rule 4). Every replicated property and every RPC ships with a one-line authority explanation the day it's written — the habit starts with the first one. The trap this week: substituting input for output. You'll know you're in it when a second tutorial starts before AResourceNode compiles. The move: build S1 first and read with a bug in hand — the shelf pulls below are for questions, not preparation.

The sessions

Five sessions, one actor and its consequences: a fact, a request, a break, a carry, and a proof with juice on top. Each item carries its done-when. The fleet's standing job all week: first entries into UE-GOTCHAS.md as they're earned — this week reliably earns some.

S1 — The fact

[NICK — by hand] Hand-write AResourceNode: a Health property declared UPROPERTY(ReplicatedUsing=OnRep_Health), registered in GetLifetimeReplicatedProps, every mutation guarded by HasAuthority(). Give the server a debug key that ticks health down, and watch both client windows react through OnRep_Health. The point of the session is the feeling: you pressed a key on the server, and the world told the clients. Why can't a client just decrement the health itself? → concept-netcode.html#authority. Done when: a server-side health change renders on both clients, and the property carries its one-line authority explanation.

S2 — The request

[NICK — by hand] The first client→server request of the project — and both of the week's silent failures live in this session, so build it with the links open. First, the home: Server_StartHarvest and Server_StopHarvest live on your Character or PlayerController — anywhere on the owned spine — and take the node as a parameter, never sitting on the node itself, because a Server RPC called on an actor the client doesn't own is dropped with no error, no warning, no log line. Where may a Server RPC live, and why does the wrong answer silently do nothing? → concept-netcode.html#ownership. Second, the shape: the pair is Reliable and fires on key press and key release, and the server ticks the drain in between — never an RPC per held frame, which is a buffer overflow on a timer, paid out as mystery disconnects. Why events, never streams? → concept-netcode.html#reliability. Write the validation — node alive, player in range — like the client is lying to you, because someday one will be. Hold-to-channel is the harvest verb for now; whacking proper arrives in week 2, when melee exists. Done when: holding the key drains the node server-side, releasing stops it, and a request from across the map does nothing.

S3 — The break

[NICK — by hand] Depletion to zero breaks the node, and the break pays the week's first physical dividend: a chunk pops out with a small launch impulse, settles, sleeps, and goes net-dormant — the bounded Tier-2 life every loose chunk in this game lives, start to finish. Respawn the node with FTimerManager. What exactly is a chunk allowed to do between pop and sleep? → doctrine-physics.html#chunk-contract. Done when: the node breaks, pops one chunk that comes to rest still on both clients, and respawns on its timer.

[FLEET] A second resource variant copied from your node pattern — the point is proving the pattern replicates, in both senses. Alongside it, SFX import scripting and CC0 sound sourcing arrive ahead of S5.

S4 — The carry

[NICK — by hand] Server_Grab attaches the chunk to your character — replicated attachment, Tier 2, no rigid-body drama — and the carry brings the waddle: a replicated CarriedWeight and a GetMaxSpeed() override on both sides, so the client predicting your movement and the server correcting it agree about how slow greed makes you. Deposit at a collection crate, count the haul with a debug counter, and take drop-on-command as the freebie it is. Why must both sides compute the same max speed? → doctrine-physics.html#cmc. Done when: carrying visibly slows you on every screen, and a chunk dropped by one client deposits from the hands of another.

S5 — The proof and the juice

[NICK — by hand] Wire the node, the pop, and the deposit into UJuiceComponent v0: floating text, a cha-ching, confetti. Sort the channels as you wire — the confetti and the cha-ching ride moments, fired once and free to miss, while the node's health and the chunk on the ground ride facts. Which channel does a visible thing belong on? → concept-netcode.html#facts-vs-moments. Then run the exam below — personally, that's the rule — and switch on net emulation at 100 ms just to feel what the round-trip costs; it's permanently on from week 3. Friday Clip #002 is the waddle. Done when: the cheater test fails safe, the juice fires on both clients, and the clip is posted.

[FLEET] JuiceKit widget boilerplate and the floating-text pool arrive before the session — S5 is wiring, not widget plumbing.

⚑ Fun payload — the JuiceKit is born

Player: when the node breaks, a physical chunk pops out with a little launch impulse and a confetti burst — and the payoff fifteen years of LitRPG promised you: floating combat text, +3 Ferrocrystal! drifting up in mono type. Then you grab the chunk and do the waddle — carrying visibly slows you, and a capsule-person waddling a comically large rock to a crate is charm at Tier-2 prices. Numbers going up, loudly, plus physical comedy: a complete game. Builder: the feedback is UJuiceComponent — one reusable component (floating text, sound stinger, particle pop, a hitstop hook for later), and every mechanic you ever build gets plugged in the day it's born. Fun engine #1, and also just good architecture. What does the JuiceKit grow into? → fun-engines.html#juicekit

What this week teaches

Three ideas, all taught by one rock. The deep versions live on the concept pages; the two silent failures are linked from the exact sessions where they bite.

The model. The node's health lives on the server; clients hold a copy and a hook — OnRep_Health — that fires when the copy changes, which is where the crack VFX, the UI tick, and the JuiceKit pop all hang. Every verb this game will ever have is this same shape again: a request goes up, a fact comes down. What does one harvest look like end to end, wire and all? → concept-netcode.html#authority. The server changes facts; clients react to facts changing.

Facts and moments. The JuiceKit is born riding the second of the wire's two channels: moments — fire-and-forget narration a late joiner is allowed to miss. The node's health and the chunk on the ground are facts, guaranteed to converge on every client that ever looks. The sorting question for everything visible you build from now on — would a join-in-progress client need to see it? — is taught at concept-netcode.html#facts-vs-moments.

The exam. The proof of internalization is the cheater test, and you run it yourself: from a client, call the harvest logic locally — a debug key wired straight to the C++, bypassing the RPC entirely. If the world changes, you built a single-player game with extra steps; if nothing happens because the client had no authority, you now understand Unreal networking better than most people with forty tutorial-hours. From here it's a habit, not a ceremony — the full procedure, run on every server-authoritative verb you ever add, lives at concept-netcode.html#cheater-test.

⚠ Gotcha seeds
  • Forgot GetLifetimeReplicatedProps → the property compiles, runs, and silently never replicates → the trace.
  • An RPC does nothing — no error, no log → check ownership first, always → concept-netcode.html#ownership.
  • A reliable RPC per held frame = buffer overflow = mystery disconnects → concept-netcode.html#reliability.
  • OnRep_* never auto-fires on the server or listen host for its own changes — call the handler manually after changing the value → concept-netcode.html#repnotify.
  • Chunks vibrating on clients = you skipped the sleep step. A chunk at rest is furniture, not a simulation → doctrine-physics.html#chunk-contract.
  • Editing a header with Live Coding active = weirdness. Header changes mean a full restart; accept it early.
